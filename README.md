# 伺か再現プロジェクト

2000年代に流行した「伺か」の体験を、ChatGPTを使った現代のmacOSデスクトップ常駐コンシェルジュとして再現するプロジェクトです。

このプロジェクトの開発はGitHubリポジトリ上で進めます。

## 目指す体験

- 人型のメインキャラクターAを右、マスコットBを左に配置してデスクトップに常駐する
- 時報や「今日は何の日」を知らせる
- 放置中に定期的な掛け合いで小噺を始める
- 吹き出しから直接検索する
- ブラウザや指定アプリを起動する
- キャラクター設定プロンプトだけで掛け合いや応答を生成する
- 喜怒哀楽などの表情差分をテキストに合わせて切り替える
- 共通立ち絵に顔・手・アイコン差分を重ねるレイヤー式スプライトに対応する

## プロジェクトで実現したいこと

最終的な目的は、複数のアプリやWebサービスに分散している情報を、デスクトップ常駐キャラクターへ集約することです。単なるChatGPTクライアントではなく、必要な情報を収集し、重要度を判断し、AとBの会話として自然に伝える「デスクトップコンシェルジュ」を目指します。

### 主要ユースケース

| 分類 | 要件 |
| --- | --- |
| ChatGPTインターフェース | 吹き出しの共通入力欄から質問し、OpenAI APIを利用してAを中心に回答する |
| API未使用時の代理機能 | ChatGPTのWebページまたはデスクトップアプリを開き、入力文をクリップボードへ渡す。Web画面のスクレイピングや回答の無断取得は行わない |
| Gmail | 新着メール、重要メール、返信が必要と思われるメールを通知・要約する |
| SNS | XなどのSNSについて、公式API、RSS、OS通知など利用可能かつ許可された手段で新着を表示する |
| YouTube | 登録または指定したチャンネルの新規動画、プレミア公開、ライブ配信開始を知らせる |
| ニュース | RSS、Googleニュース等の利用可能なフィードから関心分野の記事を定期取得し、重複を除いて紹介する |
| カレンダー | 直近の予定、開始前の予定、時間変更などを通知する |
| ローカル操作 | ブラウザ検索、アプリ起動、URLを開く、クリップボード連携などをMac上で実行する |
| キャラクター会話 | 事前登録した話題と、APIで生成した話題の両方を使ってA/B間の掛け合いを行う |

外部サービスの仕様、料金、API提供状況は変更されるため、特定サービスへ密結合せず、差し替え可能なプラグインとして実装します。

## キャラクターA/Bの役割

- **キャラクターA**: メインコンシェルジュです。デフォルトでは画面に向かって右側に配置し、ChatGPT応答、要約、検索結果、操作結果、重要通知など、実用上必要な情報を担当します
- **キャラクターB**: 通知補助・掛け合い役です。デフォルトでは画面に向かって左側に配置し、短い補足、相づち、感想、軽いツッコミ、見落とし防止の注意喚起などを担当します
- 重要情報はAの発言だけでも理解できるようにし、Bだけに必要情報を持たせません
- 通知元プラグインはA/Bのどちらに話させるかを提案できますが、最終的な発言順と表示はアプリ本体が制御します

## キャラクターと吹き出しの基本表示要件

- AとBはそれぞれ喜怒哀楽、通常、睡眠など複数の表情差分を持ち、会話内容に応じて個別に表情を変更します
- AとBの表示サイズは、それぞれ個別に変更できます
- AとBの配置位置は、それぞれ個別にドラッグ移動でき、保存した位置を次回起動時に復元します
- キャラクター透明度はA/B共通の設定値とし、1本のスライダーで同時に変更します
- AとBにはそれぞれ独立した吹き出しを表示します
- 吹き出しサイズはA/B共通の設定値とし、1本のスライダーで同時に変更します
- 吹き出し背景透明度はA/B共通の設定値とし、1本のスライダーで同時に変更します
- 吹き出し文字透明度はA/B共通の設定値とし、1本のスライダーで同時に変更します
- 吹き出しの文字サイズ、フォント、表示時間もA/B共通設定とします
- 掛け合い頻度、表示設定、スキン設定は、キャラクターの右クリックメニューから設定画面を開いて変更します
- デフォルト配置はAが右、Bが左ですが、ユーザーが移動した後は保存位置を優先します

## プラグイン受け皿の要件

Gmail、YouTube、RSS、ニュース、SNS、カレンダー、ローカル操作、AI会話などを個別に画面へ組み込まず、共通のプラグイン契約を通して追加できる構造にします。

### 共通契約

- `PluginManifest`: プラグインID、名称、バージョン、能力、必要権限、標準確認間隔を宣言する
- `CompanionEvent`: メール、動画、ライブ、ニュース、予定などを共通の通知イベントへ変換する
- `EventSourcePlugin`: 外部サービスやローカル情報から新着イベントを取得する
- `CommandPlugin`: ユーザー入力や操作命令を処理する
- `ConversationContextPlugin`: ChatGPTへ渡す追加コンテキストを提供する
- `PluginRegistry`: 登録済みプラグインと有効・無効状態を管理する
- `EventPipeline`: 重複排除、重要度判定、静音時間、即時通知、ダイジェスト送りを制御する

詳細仕様は [`docs/PLUGIN_ARCHITECTURE.md`](docs/PLUGIN_ARCHITECTURE.md) を参照してください。

### 安全性と権限

- APIキー、OAuthトークン、更新トークンは平文ファイルやUserDefaultsへ保存せず、macOS Keychainへ保存する
- プラグインは必要な権限をマニフェストで宣言し、アプリ本体が許可状態を管理する
- プラグインからUIを直接操作させず、共通イベントと共通アクションを返させる
- 取得失敗、認証切れ、レート制限はプラグイン単位で隔離し、アプリ全体の停止を避ける
- 同じメール、動画、記事を繰り返し通知しないよう、重複排除キーと取得履歴を保持する
- 静音時間、通知頻度、重要度、サービスごとの有効・無効をユーザーが設定できるようにする
- 外部へ送信する情報と送信先を設定画面で確認できるようにする

### プラグイン配布方針

初期段階では、リポジトリ内で実装し、アプリと同時にビルドする内蔵プラグインを正式対応とします。

署名されていない第三者コードをアプリ内へ直接読み込む方式は採用しません。任意コードの直接ロードは、Keychainやファイルへの不正アクセス、クラッシュの波及、macOSの署名・公証との衝突を招くためです。第三者プラグインを配布可能にする段階では、XPCまたは別プロセスで隔離し、権限と通信形式を制限します。

- macOS 14 Sonoma以降

## インストール

GitHub Actionsの `Build and test` からmacOS成果物を取得できます。ZIPを展開してアプリを `Applications` フォルダへ移すか、DMGを開いてアプリを `Applications` へドラッグします。

正式なDeveloper ID署名とApple公証を行う前のベータ成果物では、初回起動時にFinderでアプリを右クリックして「開く」を選ぶ必要がある場合があります。

ソースからローカル用アプリを作る場合:

```bash
Scripts/build-app.sh
Scripts/verify-app.sh
Scripts/install-local.sh
```

開発用に直接起動するだけなら `swift run "伺か再現プロジェクト"` も利用できます。APIキーは初回設定または設定画面で入力し、macOS Keychainへ保存します。

## 入力コマンド

- `検索: キーワード` でブラウザ検索
- `起動: Safari` のように入力してアプリ起動
- `今日は何の日` で内蔵の記念日情報を表示
- 通常の文章はキャラクターへの相談として扱う

## メニューと設定

- メニューバーから表示、非表示、設定、再起動、終了を実行できます
- デスクトップ上のキャラクターを右クリックしても同じ基本メニューを開けます
- 設定画面ではAPI接続、キャラクターの性格、モデル画像、A/B個別サイズ、A/B共通のキャラクター透明度、吹き出し背景透明度、吹き出し文字透明度、フォント、独り言、クリック透過、常時前面、ログイン時起動を変更できるようにします
- 設定画面の編集内容は「設定を保存」を押すまで反映されません。APIキーも保存時だけKeychainへ書き込みます
- APIキー入力後に利用可能モデルを取得し、選択したモデルで接続テストを行います。既定モデルは `gpt-5-mini` です
- AIによる自動独り言は初期OFFです。有効化時も30から60分間隔、日次上限付きで、スリープ、画面ロック、フルスクリーン中は停止します
- 吹き出しは一定時間後に閉じ、ウィンドウもキャラクターだけの大きさへ縮みます

## 設定ライフサイクル

- 初回起動: 初回設定画面を表示し、APIキー、選択式モデル、キャラクター名、性格プロンプト、ログイン時自動起動を設定できます
- 通常設定: メニューバーまたは右クリックメニューから設定画面を開き、保存ボタンを押した時だけ変更内容を反映します
- APIキー: macOS Keychainに保存します。旧バージョンのUserDefaultsにあるキーは初回読み込み時に自動移行します
- 終了時: アプリ終了または再起動の直前に設定値を同期します
- 次回起動: UserDefaultsから保存済み設定を読み込み、キャラクター、画像フォルダ、透過度、独り言間隔、ChatGPT連携を復元します
- 常駐運用: 設定画面の「ログイン時に自動起動」を有効にすると、macOSのログイン項目として登録します。承認が必要な場合は設定画面からシステム設定を開けます

## アンインストール

メニューバー、右クリックメニュー、または設定画面から「アンインストール」を実行します。確認後、次をまとめて処理します。

- macOSのログイン時起動を解除
- UserDefaultsのアプリ設定を削除
- KeychainのOpenAI APIキーを削除
- アプリ本体をゴミ箱へ移動

設定画面で指定した外部モデル画像フォルダはユーザー所有データとして削除しません。

## OpenAI APIへ送信する情報

APIキー設定時は、入力した文章、直近のユーザー発言とキャラクター発言、2名の名前と性格プロンプト、指定モデル名をOpenAI Responses APIへ送信します。検索語とアプリ起動名はOpenAIへ送らず、Mac上で直接処理します。

- Responses APIリクエストには `store: false` を指定し、レスポンスのアプリケーション状態を保存しない設定にします
- OpenAIの不正利用監視ログは、通常のAPI利用ではプロンプトや応答を含む場合があり、既定で最大30日保持されることがあります。利用組織でZero Data Retention等が承認されている場合はその組織設定が適用されます
- APIキーと会話はこのアプリの開発者サーバーを経由せず、MacからOpenAI APIへ直接送信します
- API利用料金はユーザー自身のOpenAI APIアカウントに発生します。ChatGPT Plus等の料金には含まれません
- AIによる自動独り言は初期OFFで、設定画面に当日のAPIリクエスト回数を表示します

## キャラクター画像

画像は `Sources/UkagakaReproductionProject/Resources/Characters/` に置きます。
配布後に別モデルへ差し替える場合は、設定画面の「モデル画像」で同じ構造の外部フォルダを指定します。
外部フォルダが指定されている場合はそちらを優先し、足りない画像はアプリ同梱画像へフォールバックします。

レイヤー式の画像がある場合は、そちらを優先して読み込みます。全レイヤーは同じキャンバスサイズ、同じ原点、透過PNGで書き出してください。

```text
    face_fun.png
    face_sleep.png
    hand_default.png
    hand_wave.png

    hand_think.png
    hand_emphasize.png
    hand_sleep.png
    icon_happy.png
    icon_angry.png
    icon_sad.png
    icon_fun.png
    icon_sleep.png

  character_b/
    base.png
    face_happy.png
    face_angry.png
    face_sad.png
    face_sleep.png
    icon_sad.png
    icon_sleep.png
```

### 推奨モデル画像仕様

- 形式: 透過PNG、sRGB、ファイル名は半角英数字・小文字・アンダースコアのみ
- 基準サイズ: 立ち絵キャラは縦1600から2400px程度、ぬいぐるみは縦1000から1600px程度
- キャンバス: 1キャラクター内の全レイヤーで幅・高さ・原点・足元の基準線を完全にそろえる
- 余白: 上下左右に8から12%程度の透明余白を残し、手やアイコンが切れないようにする
- 解像感: アプリ上では標準で高さ155から225px程度、設定により50から150%で表示されるため、縮小後も輪郭線と表情パーツが読める太さにする
- 背景: 完全透過にし、白背景や影を焼き込まない。影が必要な場合は将来 `shadow.png` のような独立レイヤーにする
- 色: キャラクター本体は同じパレットで統一し、表情差分で色味や線幅が変わらないようにする
- レイヤー順: `base.png`、`hand_*.png`、`face_*.png`、`icon_*.png` の順に合成される
- `base.png`: 体、髪、服、固定小物など、表情や手の動きで変わらない部分
- `hand_*.png`: 腕や手だけを含む差分。不要なモデルでは省略可能
- `face_*.png`: 眉、口、目元などの表情差分。ぬいぐるみは瓶底眼鏡を崩さず、目が見えない前提を維持する
- `icon_*.png`: 涙、怒りマーク、`Zzz` など、顔を大きく変えずに感情を補う小物
- 容量目安: 1枚あたり5MB以下、1モデルパック全体で50MB以下を目標にする

配布後に差し替えるモデルパックは、任意の場所に次のような構成で置き、設定画面からそのフォルダを選択します。

```text
MyModelPack/
  character_a/
    base.png
    face_happy.png
    face_angry.png
    face_sad.png
    face_sleep.png
    hand_default.png
    hand_wave.png

  character_b/
    base.png
    face_angry.png
    face_sad.png
    face_sleep.png
    icon_sad.png
    icon_sleep.png
```

ぬいぐるみ側は喜の構図を `character_b/base.png` として固定し、怒は眉と口、哀は左レンズの水滴、寝は `Zzz` などのアイコンだけで表現する想定です。

レイヤー式画像がない場合は、従来の1枚絵ファイルへフォールバックします。

- `character_a_neutral.png`
- `character_a_happy.png`
- `character_a_angry.png`
- `character_a_sad.png`
- `character_a_fun.png`
- `character_a_sleep.png`
- `character_b_neutral.png`
- `character_b_happy.png`
- `character_b_angry.png`
- `character_b_sad.png`
- `character_b_sleep.png`

画像がない場合は仮のシルエット表示になります。
読み込んだ画像は更新日時とファイルサイズを含むキーで事前デコード・キャッシュし、モデル画像フォルダを変更した時にキャッシュを破棄します。

## 次回変更案: キャラクター別吹き出し

現在はAとBの発言を1つの共通吹き出しにまとめています。次回変更では、AとBからそれぞれ独立した吹き出しを表示し、役割を明確に分離します。

### 会話上の役割

- キャラクターAはメインコンシェルジュとして、ユーザーへの回答、検索・アプリ起動結果、時報、今日の情報、エラーや操作案内を担当する
- キャラクターBは掛け合い役として、Aへの相づち、感想、短い補足、軽いツッコミや注意喚起を担当する
- 実用上必要な情報はAの発言だけで完結させ、重要情報をBだけに含めない
- 通常のユーザー応答は原則としてAを1件目、必要な場合だけBを2件目にする
- 検索、アプリ起動、エラー通知などの操作結果は原則としてAだけが発言する

### 表示仕様

- Aの吹き出しはAの頭上に大きめに表示し、吹き出しの尻尾をAの位置へ向ける
- Bの吹き出しはBの頭上に小さめに表示し、吹き出しの尻尾をBの位置へ向ける
- Aだけが発言した場合はAの吹き出しだけ、AとBが発言した場合は2つを同時表示する
- 入力欄はキャラクター別に分割せず、アプリ全体に対する共通入力欄を1つ維持する
- 吹き出しの収納は当面共通タイマーとし、A・Bを同時に閉じる
- 吹き出しサイズ、背景透明度、文字透明度、文字サイズ、フォント、表示時間はA/B共通設定とする
- 398px前後の現行パネルで文章が狭くなりすぎないよう、AとBの吹き出しは完全な左右並列ではなく上下方向に少しずらして配置する

### 実装方針

- 現在の `SpeechBubbleView` を、1キャラクター・1発言を描画する `CharacterSpeechBubbleView` 相当へ分割する
- `CharacterLine.speakerID` からA用・B用の最新発言を抽出し、キャラクターごとに描画する
- `SpeechBubbleShape` の尻尾位置を左・中央・右、またはキャラクター中心座標に応じて変更できるようにする
- ローカルコマンドの発言者指定で `characters.first` に依存せず、`character_a` を明示する
- OpenAIへの開発者プロンプトにAとBの役割分担、A優先、Bの短文化を追加する
- 生成結果がBから始まった場合にAを先頭へ補正し、重複発言やBだけの主要回答を防ぐ
- 非発言側の表情・ジェスチャーを必要に応じてニュートラルへ戻す

### 初回実装の対象外

- AとBの吹き出しを別々のタイマーで閉じる機能
- 発言を時間差で逐次表示する演出
- Bの掛け合い頻度を設定画面から変更する機能
- キャラクターごとに別の入力欄を持たせる機能

これらはキャラクター別吹き出しの基本動作が安定した後に検討します。
